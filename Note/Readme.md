# 项目总体搭建笔记
## docker篇
docker中，`Dockerfile`只用于构建docker image，它是在`docker build`时起作用的。
若是要配置开发环境，需要进行**挂载**操作，则是在`docker run`时进行指令添加的，但是这样往往十分的麻烦。
若是要实现挂载操作的自动化，就需要借助`docker-compose.yml`，来管理开发环境的挂载配置，同时，它还可以管理其他的配置，如端口映射、环境变量等。除此之外，还能够进行多个容器的统一管理。

> 借助vscode的`dev container`插件，直接实现了开发环境的自动化配置，无需再配置docker-compose.yml等复杂操作了。
