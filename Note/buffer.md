# Buffer实现时所学

## size_t/ssize_t/uintxx
这几种类型总是让人傻傻分不清，其实他们之间是有明显的差别和较为固定的用法的。

### size_t
对于size_t其实应该不是很陌生了：在`std::vector`中，在使用`size()`的时候，其返回值就是size_t类型的。

size_t它是平台相关的，它的实现是根据ABI走的，因此它跟机器字长紧密关联，在64位机器上，它通常是64位无符号整数，而32位机器上则是32位无符号整数。它的作用是：**对其标准库语义，避免负数长度问题**。因此，在某些场景下基本都是使用size_t，如：数组的index，用来表示某个对象/内存的大小等。

size_t的出现原因是因为，`sizeof()`能够表示任何对象的大小，因此它需要一个专门的类型，能够随平台的指针/地址空间的变化而变化。

### ssize_t
`ssize_t`是POSIX标准下的，**有符号版本的**`size_t`。它主要用于read()/write()这种系统调用，它们统一规定使用`-1`表示错误，同时还需要能够表示操作了多少字节。

在长度上，`size_t`和`ssize_t`是等长的，但是由于ssize_t需要表示负数，因此它的正数上限是要小于size_t的。

### cstdint的uintxx_t
这种主要是机器类型，属于固定宽度的整数，主要用于协议字段中。它没有歧义也不存在平台差异，对宽度是“保证”的。
